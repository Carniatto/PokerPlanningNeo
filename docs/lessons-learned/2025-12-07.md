## Lessons Learned
1.  **CSS vs. SVG for Text Effects**: When a specific, high-fidelity text effect like "Gradient Outline + Transparent Fill" is requested, **SVG** is the superior tool. CSS `background-clip: text` works for *filled* gradients, but combining it with `text-stroke` for an outline often leads to rendering artifacts or poor browser support. SVG offers precision control.
2.  **Simplicity Wins**: While "gamified" elements like animations and status dots can seem appealing, they often add cognitive load. Users frequently prefer a "clean" UI where status is conveyed through subtle cues (like a single checkmark) rather than multiple redundant indicators. Always be ready to simplify.
3.  **Process Management matters**: When tools (like Firebase) fail with "Port already in use," it is crucial to check for and kill zombie processes rather than just retrying or changing ports blindly.
4.  **Absolute Paths in Artifacts**: To ensure images verify correctly in user-facing artifacts, always use **absolute file paths**. Relative paths often fail to render in the preview environment.
