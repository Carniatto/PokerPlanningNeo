# Product Requirements

## Current Features (Implemented)
-   [x] **Room Creation:** Users can create a room and become Host.
-   [x] **Joining:** Users can join via code or direct link.
-   [x] **Identity:** Users enter a name (persisted per session?).
-   [x] **Voting:** Standard Fibonacci deck + ? + Coffee.
-   [x] **Host Controls:** Reveal votes, Reset votes.
-   [x] **Real-time Updates:** Participant list, vote status, card reveal.
-   [x] **Responsive UI:** Basic responsive layout.

## Missing / Planned Requirements (To be prioritized)

### High Priority
-   [ ] **Spectator Mode:** Allow users to join without voting (e.g., PO, Stakeholders).
-   [ ] **Vote Distribution:** Visual summary of votes (e.g., bar chart) after reveal to quickly see consensus/outliers.
-   [ ] **Topic/Story Name:** Ability for Host to set the name of the story currently being voted on.
-   [ ] **Kick User:** Host ability to remove a user (ghosts/trolls).

### Medium Priority
-   [ ] **Custom Decks:** T-Shirt sizes (XS, S, M, L, XL), Modified Fibonacci.
-   [ ] **Timer:** Optional countdown for voting.
-   [ ] **Emoji Reactions:** Fun interactions during voting.

### Low Priority / Future
-   [ ] **Jira/Linear Integration:** Import tickets directly.
-   [ ] **Export Results:** Download CSV of voting history.
-   [ ] **User Accounts:** Optional login to save history/settings.
